# Чеклісти для тестування

## Модуль: Модуль: Відображення розкладу для обраного викладача (публічна частина)

### Функціональність (позитивні сценарії)

 - [ ] Відображається список викладачів у полі пошуку / вибору

 - [ ] Після вибору викладача відображається його розклад

 - [ ] Розклад містить коректні дні тижня (Пн–Пт)

 - [ ] Для кожного дня відображаються пари із актуальними даними (номер пари, час, аудиторія, предмет)

 - [ ] Відображаються лише ті дні, у які є заняття

 - [ ] Якщо викладач має кілька пар в один день — вони відображаються у правильному порядку

 - [ ] Дані оновлюються після вибору іншого викладача

 - [ ] Дані оновлюються миннєво 

### Негативні сценарії
 - [ ] При відсутності вибору викладача розклад не відображається або показується повідомлення

 - [ ] При пошуку неіснуючого викладача система показує коректне повідомлення (наприклад, "Викладача не знайдено")

 - [ ] Якщо у викладача немає занять — відображається відповідне повідомлення

 - [ ] Система не падає дані при відсутності даних у базі

 - [ ] При миттєво ( до 2 секунд) перемиканні між викладачами дані не змішуються

 - [ ] Якщо виникає помилка завантаження даних — відображається повідомлення про помилку

 - [ ] Некоректний параметр у URL (якщо є) не викликає помилки сторінки

### UI/UX
- [ ] Таблиця розкладу має чітку структуру (дні, пари, час)

- [ ] Заголовок сторінки містить ПІБ обраного викладача

- [ ] Поле пошуку викладача розташоване з верху над розкладом де його добре видно, при вводі з символів підтянуються схожі дані

- [ ] Інтерфейс не "ламається" при великій кількості пар ( накдадання тексту , накладання пар або навпарки зникання )

 - [ ] Інтерфейс коректно відображається на різних розмірах екрану

 - [ ] Немає накладання тексту або зсувів таблиці

 - [ ] Час і номер пари написані великими буквами і іншим кольроб для легкого чинання 

 - [ ] Є візуальне розділення днів тижня

 - [ ] Парний/непарний тиждень розташовані один коло одного на одній сторінці
